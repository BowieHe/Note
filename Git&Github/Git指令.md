# 基本操作

1. `git status` 查看工作区，暂存区状态

2. `git add [filename]`将工作区的“新建/修改”添加到暂存区

3. `git commit -m "message" [filename]`将暂存区的文件递交到本地库

4. `git log`查看提交历史
   空格向下翻页，b向上翻页，q退出

   1. `git log --pretty=online`按照制定格式输出日志，默认为medium，这里为online，raw则会显示出每次提交的更多信息

   2. `git log --online`将每天日志的输出为一行

   3. `git -[length]`指定显示多少条日志
      `git log --skip=[skip]`指定跳过前面几条日志

   4. `git reflog`用于查看git的提交记录，如果一个brunch的提交记录被误删，可以通过`got reflog`里面的SHA1校验和来恢复
      其中`提交记录简写 HEAD@{移动到当前版本需要多少步}`

   5. `git reset`版本前进和后退

      1. `git reset --hard [局部索引值]`
      2. `git reset --hard HEAD^`后退一步，一个^表示一步
      3. `git reset --hard HEAD~n`后退n步

      三个参数对比

      --soft

      + 只在本地移动HEAD指针

      --mixed

      + 在本地库移动HEAD指针，重置暂存区

      --hard

      + 在本地库移动HEAD指针，重置暂存区，重置工作区

   6. `git diff [filename]`比较文件差异

# 分支管理

1. 分支好处
   1. 同时并行推荐多个功能开发，提高开发效率
   2. 各个分支在开发过程中，如果某一个开发失败，不会对其他分支有任何影响

2. 操作指令
   1. `git branch [分支名]`创建分支
   2. `git branch -v`查看分支
   3. `git checkout [分支名]`切换分支
   4. 合并分支
      1. 切换到被合并的分支，`git checkout [被合并分支名]`
      2. `git merge [有新内容的分支名]`

## Git基本原理

1. 哈希算法

   1. 哈希算法是一系列的加密算法，不管输入数据量多大，输入同一个哈希算法，得到的加密长度固定
   2. 哈希算法确定，输入数据确定，输出数据能够保持不变
   3. 算法确定，输入数据有变，输出数据一定有变化，通常变化很大
   4. 哈希算法不可逆

   Git采用的就是SHA-1算法
   通过原始文件和经过网络传输后的目标文件的SHA-1值进行比较，如果不一致则表示目标文件损坏

2. 文件管理机制（快照）

   每个小文件都有自己的哈希值，然后上传之后会产生一个树，每个字文件在树里面是值的形式存在（保留原哈希值）。上传之后，commit会有对应的哈希值。

   不同版本会有不同的哈希值，但是如果某版本里面一个子文件没有被修改

   每一个版本会指向之前的父版本信息
